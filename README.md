# OzoneShell

A desktop shell for Arch Linux built on **Hyprland** and **Quickshell** (QML), with a hand-written TOML theming system and a Python pipeline that generates configuration for every program from a single source of truth.

Started as a personal rice, OzoneShell is evolving into a more structured, reusable framework — in the spirit of projects like [caelestia](https://github.com/caelestia-dots/caelestia).

---

## Project status

🚧 **Actively in development, not yet ready for third-party "drop-in" use.**

Some parts of the pipeline (in particular the config generation from TOML themes) were written with AI assistance to quickly test the end-to-end flow while the project structure stabilizes. These are not considered final: they will be progressively rewritten and refined by hand as the architecture solidifies. If you're browsing this repo for reference, keep in mind that some parts are still test scaffolding rather than finished code.

The TOML themes and the overall framework structure (Hyprland/Quickshell/monorepo) are designed and written entirely by me.

## Stack

- **Hyprland** — window manager, configured in Lua
- **Quickshell** (QML) — bar, widgets, shell graphical components
- **Python** — theme generation pipeline (TOML → Lua / conf / JSON)
- **TOML** — theme format, hand-written
- **kitty** — terminal, themed by the same pipeline

## Repo structure

```
OzoneShell/
├── install/
│   └── bootstrap.py       # hardware detection (monitors, battery) + symlinks to ~/.config
├── scripts/
│   └── setTheme.py         # applies the active theme: TOML -> lua/conf/json
├── themes/
│   ├── pansy/theme.toml
│   ├── lumen/theme.toml
│   └── .../theme.toml
├── hypr/                   # Hyprland config (Lua) — symlinked from ~/.config/hypr
├── kitty/                  # kitty config — symlinked from ~/.config/kitty
├── shell/                  # Quickshell QML — symlinked from ~/.config/quickshell
├── cava/                   
├── fuzzel/
├── fastfetch/
└── state/                  # runtime-generated files (gitignored)
```

Each program folder (`hypr/`, `kitty/`, `shell/`) contains a `state/` subfolder with the config fragments generated from the active theme — these files are **not committed**, they're regenerated by `setTheme.py` on every theme change.

## Installation

```bash
git clone <repo-url> ~/OzoneShell
cd ~/OzoneShell
python install/bootstrap.py
```

This will:
1. Detect the number of connected monitors (for workspace mode) and whether a battery is present (for wip features)
2. Create symlinks from `~/.config/*` to the repo folders
3. Write `state/system.toml` with the detected data

Then set a theme:
```bash
python scripts/setTheme.py ozone
```

## Theming

Themes are written in TOML under `themes/<name>/theme.toml`. Each theme defines a color palette (`normal`/`bright`/`special`), terminal appearance, fonts, and Hyprland-specific parameters (gaps, borders, gradients).

Gradients reference other colors from the same theme via a dotted path, e.g.:
```toml
active_gradient = ["colors.normal.purple", "colors.bright.black"]
```

`setTheme.py` resolves these references and generates the configuration files for each program.
